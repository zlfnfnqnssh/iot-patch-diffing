# P-015

- **status**: active (v1)
- **공식**: `rpc_arg` → `length_bound` 부재 → `stack_buffer_copy`
  - source_detail=`SNR plane id + sensor meta struct`, sink_detail=`strcpy(a3+24, 32-byte stack src)`
- **severity_hint**: medium
- **origin**: `mhal.ko/MHal_SNR_GetPlaneInfo`  (언어: decompiled_c)

## 요약

Sensor plane id 로부터 얻은 32-byte 스택 버퍼를 길이 검사 없이 strcpy 로 호출자 구조체에 복사. BinDiff 매칭 0.24 로 불확실.

## 상세 설명

커널 모듈 mhal.ko 의 센서 플레인 정보 getter. 32바이트 스택 로컬(src/v16) 을 호출자 a3+24 에 strcpy 로 복사하므로 src 가 non-null 종료 상태면 OOB read/write 가능. OLD/NEW 의 strcpy 형식이 동일해 실제 '패치' 여부를 특정할 수 없음(BinDiff sim=0.24 오매칭 의심). 사람 검토 필요: 다른 함수로 재매칭되거나 NEW 버전에 길이 검사가 있는지 확인.

## 취약 코드 (OLD)

```c
char src[32];   // initialized via DrvSensorGetPlaneInfo
if (a1 > 3 || DrvSensorGetPlaneInfo()) return -1;
strcpy((char *)(a3 + 24), src);
```

## 수정된 코드 (NEW)

```c
_BYTE v16[32];
if (a1 > 3) return -1;
if (DrvSensorGetPlaneInfo()) return -1;
strcpy(a3 + 24, v16);   // 길이 경계 여전히 없음
```

## 공격 시나리오

인증된 로컬 프로세스가 ioctl 로 sensor pad id 를 지정하면 커널이 플레인 메타를 스택 버퍼에 채우고 strcpy 로 사용자 구조체(a3)에 복사. a3+24 이후에 다른 필드가 있거나 src 에 NUL 종료가 없으면 인접 메모리 손상 가능.

## 수정 설명

현재 diff 에서는 길이 경계가 도입된 증거 없음. 재-매칭 후 재검토 필요.
