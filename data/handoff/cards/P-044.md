# P-044

- **status**: active (v1)
- **공식**: `ioctl_arg` → `range_check` 부재 → `array_index_write`
  - source_detail=`gpio number a1 (unsigned int)`, sink_detail=`LANCHOR0_3[52 * (a1 >> 3) + …] = …`
- **severity_hint**: medium

## 요약

GPIO 번호의 bank 상한(>0x5F) 검사가 NEW에서 제거되어, 외부 a1 값으로 정적 배열 LANCHOR0_3에 OOB write 가능.

## 상세 설명

OLD 버전은 함수 초입에서 `a1 > 0x5F`로 GPIO bank 인덱스를 0x5F 이하로 제한한 뒤 배열 접근을 수행. NEW에서는 이 검사가 함수 본체에서 사라져 a1 값에 따라 `LANCHOR0_3[52 * (a1>>3) + offset]`의 쓰기 위치가 경계를 넘을 수 있음. 상위 호출자에서 검사가 이루어지는지 확증되지 않음.

## 취약 코드 (OLD)

```c
// NEW: bank range check 제거
v2 = a1 >> 3;
v3 = 52 * (a1 >> 3);
v4 = a1 & 7;
v7 = 1 << (a1 & 7);
OSA_spinLock(LANCHOR0_3[v3 + 10]);
// 이후 LANCHOR0_3[v9 + 28] = a2[2]; 등으로 쓰기
```

## 수정된 코드 (NEW)

```c
// OLD: bank 상한 검사 존재
if ( a1 > 0x5F ) {
  OSA_logWriteV2(1, "prc", ..., "bank %d should small than %d", v4, 12);
  return -1;
}
v7 = &LANCHOR0_3[52 * v4];
OSA_spinLock(v7[10]);
```

## 공격 시나리오

사용자 공간에서 GPIO 인터럽트 등록 ioctl을 호출할 때 gpio 번호를 0x5F 초과로 전달 → 정적 전역 LANCHOR0_3 배열 뒤쪽 메모리에 임의 스핀락/콜백/함수 포인터 기록 → 커널 권한 상승 가능성.
