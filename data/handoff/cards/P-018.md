# P-018

- **status**: active (v1)
- **공식**: `procfs_write` → `length_check` 부재 → `stack_strcpy`
  - source_detail=`debugfs dump_dram cmd args`, sink_detail=`strcpy into 128-byte stack buffer without bound`
- **severity_hint**: low
- **origin**: `mi_venc.ko/_MI_VENC_ProcDumpDram`  (언어: decompiled_c)

## 요약

root-only debugfs 진입점이지만 128바이트 스택 버퍼 2개에 외부 문자열을 strcpy로 무제한 복사. 2.840 버전도 동일 패턴 유지.

## 상세 설명

ProcDumpDram은 /proc debugfs 명령 처리기로 a5+8/a5+16에서 사용자 문자열을 받아 로컬 128바이트 스택 버퍼에 strcpy한다. 길이 검사 없음. OLD/NEW 모두 동일 strcpy 사용. 공격 벡터는 root 권한 필요(local privilege escalation 불가).

## 취약 코드 (OLD)

```c
char s[128];
char v34[128];
memset(s, 0, sizeof(s));
memset(v34, 0, sizeof(v34));
strcpy(s, *(const char **)(a5 + 8));
strcpy(v34, *(_DWORD *)(a5 + 8));
```

## 수정된 코드 (NEW)

```c
char s[128];
strcpy(s, *(const char **)(a5 + 8));
/* NEW: 동일 패턴 유지, 로그 래퍼만 축약 */
```

## 공격 시나리오

root 컨텍스트에서 debugfs /proc 노드에 128바이트 초과 경로 쓰기 → 스택 오버플로 (canary 우회 시 루트 kernel write).

## 수정 설명

2.840에서 수정되지 않음. Hunter 단계에서 유사 패턴(strcpy + 128바이트 스택 + debugfs) 식별 목적으로 카드 등록.
