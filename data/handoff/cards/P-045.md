## 요약

외부 입력 username/password를 JSON 템플릿 snprintf에 직접 삽입하여 ubus(cloudclient/user_bind)로 전달. 따옴표/백슬래시 이스케이프 부재로 JSON 구조 변조 가능.

## 상세 설명

TP-Link Tapo dsd 바이너리의 cc_bind 엔드포인트는 클라우드 계정 연결 시 username/password를 ubus cloudclient.user_bind 메시지 본문에 JSON으로 직렬화해 넘긴다. OLD에서는 snprintf에 원문을 그대로 삽입해 따옴표/백슬래시가 JSON 경계를 파괴할 수 있었고, NEW에서는 각 바이트를 순회하며 세 종류 메타문자를 백슬래시 이스케이프한 별도 버퍼를 만든 뒤 템플릿에 삽입하도록 바뀌었다.

## 취약 코드 (OLD)

```c
v5 = jso_obj_get_string_origin(a2, "username");
string_origin = jso_obj_get_string_origin(a2, "password");
snprintf(v8, 1025, "{\"username\": \"%s\", \"password\": \"%s\"}", v5, string_origin);
sub_403600("cloudclient", "user_bind", v8);
```

## 수정된 코드 (NEW)

```c
// escape ' " \ for username/password into v22/v23
switch (c) {
  case '\'': buf[i++]='\\'; buf[i++]='\''; break;
  case '"':  buf[i++]='\\'; buf[i++]='"'; break;
  case '\\': buf[i++]='\\'; buf[i++]='\\'; break;
  default:   buf[i++]=c;
}
snprintf(v21, 1025, "{\"username\": \"%s\", \"password\": \"%s\"}", v23, v22);
```

## 공격 시나리오

인증되지 않은(혹은 인접 권한) 공격자가 username에 `a", "admin_flag": "1` 같은 페이로드를 전달하면 OLD는 {\"username\": \"a\", \"admin_flag\": \"1\", \"password\": \"...\"} 구조로 파싱되어 cloudclient 측에서 추가 필드 주입/검증 우회가 가능.

## 수정 설명

문자 단위 switch로 '/"/\\ 세 메타문자에 역슬래시를 전치해 별도 스택 버퍼(v22,v23, 각 256바이트)에 기록 후 JSON 템플릿에 치환. 길이 초과 가드는 여전히 미흡하나 경계 문자는 중립화됨.
