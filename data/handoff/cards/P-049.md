## 요약

HTTP h264 스트림 콜백에서 약한 인증 래퍼를 digest 인증 래퍼로 교체하고 device_locked 거부 체크를 새로 추가.

## 상세 설명

TP-Link Tapo C200 cet 데몬의 HTTP 기반 h264 스트림 콜백(mixed/playback)에서, loopback이 아닌 원격 요청이 들어올 때 기존 약한 인증 래퍼 sub_41BB7C가 digest 인증 래퍼 sub_41B9D0으로 교체되었고, device_locked(sub_460128) 상태에서 스트림 재생을 거부하는 분기가 추가됨. OLD에서는 loopback이나 a1[30] 플래그가 참이면 인증을 건너뛸 여지가 있었고 device_locked 체크는 vod 경로에만 존재했음.

## 공격 시나리오

원격 공격자가 Tapo C200에 loopback으로 위장하거나 User-Agent를 ipc/relayd로 속이고, 약한 sub_41BB7C가 허용하는 조건을 만족시켜 h264 스트림(영상 라이브 미리보기)을 인증 없이 가져옴. 새 패치로는 digest 인증 검증 필수이며 device_locked 상태에서는 재생 즉시 거부.

## 수정 설명
