## 요약

DNS v4 읽기 루틴을 write 측과 동일한 OSA_mutex 보호 경로에 합류시켜 중간 상태 노출을 차단.

## 상세 설명

쓰기 경로가 mutex로 보호돼도 읽기 경로가 락을 공유하지 않으면 fopen 실패 시 기본값(202.101.172.35) 반환이 경합에 의해 일어날 수 있었다. NEW는 같은 mutex 도메인에 합류.

## 취약 코드 (OLD)

```c
if ( fopen() ) {
  if ( fgets() && strstr() ) sscanf();
  if ( fgets() ) { if ( strstr() ) sscanf(); }
  fclose();
  return 0;
}
strcpy(a1, "202.101.172.35");
strcpy(a3, "202.101.172.35");
```

## 수정된 코드 (NEW)

```c
OSA_mutexLock();
if ( fopen() ) {
  if ( fgets() && strstr() ) sscanf();
  if ( fgets() ) { if ( strstr() ) sscanf(); }
  fclose();
} else {
  strcpy(a1, "202.101.172.35");
  strcpy(a3, "202.101.172.35");
}
OSA_mutexUnlock();
return 0;
```
