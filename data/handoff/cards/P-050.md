## 요약

uci 기반 media encrypt 비활성 경로에서 평문 하드코딩 키를 RSA 암호화 블롭으로 대체해 firmware strings 노출 표면 제거.

## 상세 설명

cet가 미디어 암호화 설정을 로드할 때 사용자가 ciphertext를 지정하지 않은 기본 경로에서, 과거 버전은 firmware 바이너리에 평문으로 'TPL075526460603' 키를 내장하여 strings 또는 정적 분석으로 즉시 발견 가능했음. 새 버전은 해당 상수를 RSA 암호화 Base64 블롭으로 저장하고 부팅 시 rsa_decrypt로 복원해 런타임에만 메모리에 존재하게 함.

## 취약 코드 (OLD)

```c
else
{
    v12 = "TPL075526460603";
}
strcpy(byte_4D9280, v12);
```

## 수정된 코드 (NEW)

```c
else
{
    v12 = (char *)rsa_decrypt(
              "oP4b0Q2dUvDXKwK9gLnBfVpxcHUDg4V2Wxu366l93TuJifzhtYu8NI7BeC7wvAd2e8DVv7pi5pK7ooHIPC7MuZAeU3...=",
              172);
    strcpy(byte_4DA8D0, v12);
    free(v12);
}
```

## 공격 시나리오

firmware dump를 확보한 공격자가 strings로 디폴트 미디어 키를 획득해 RTSP/HTTP 암호화 스트림을 복호화하는 시나리오. 패치 후에는 개인키/리버싱 난이도가 증가.

## 수정 설명

평문 상수 치환 → rsa_decrypt(base64_blob, 172) 호출 결과로 대체. 이후 free로 해제.
