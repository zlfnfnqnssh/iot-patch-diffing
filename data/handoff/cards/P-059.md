## 요약

boot 파라미터 조회를 NoPrint 변종으로 교체하고 복합 조건을 early-return 체인으로 분리.

## 상세 설명

조건식 중 `j_getBootParameter`는 로그로 파라미터 값을 덤프하는 것으로 추정되며, NEW는 NoPrint 변종으로 전환해 민감한 테스트 모드 값을 로그에서 제거. 별도 strncmp/OSA_fileAccess 단계로 분리되어 감사 가능성과 단락 평가 이해도 향상.

## 취약 코드 (OLD)

```c
_BYTE v1[480];
memset(v1, 0, 464);
snprintf();
if ( OSA_fileAccess()
  || j_getBootParameter("ipd_testmode", v1, 16) && strncmp() )
  return -1;
```

## 수정된 코드 (NEW)

```c
_BYTE v1[384];
memset(v1, 0, sizeof(v1));

  return -1;

  return -1;

if ( OSA_fileAccess() )
  return -1;
```
