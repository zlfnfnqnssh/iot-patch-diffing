## 요약

DNS v4 주소 쓰기 루틴 전체를 OSA_mutexLock/Unlock로 감싸 동시 수정 시 파일 손상을 차단.

## 상세 설명

두 DNS 항목을 각각 sprintf로 포맷해 fputs로 기록하는 동안 다른 쓰레드가 같은 함수를 호출하면 중간 상태에서 파일이 잘리거나 포인터가 덮어써질 수 있다. NEW는 mutex로 전체 영역을 보호.

## 취약 코드 (OLD)

```c
if ( fopen() ) {
  v0 = calloc();
  sprintf(); sprintf();
  if ( fopen() ) fputs();
  fclose();
  return 0;
}
return -1;
```

## 수정된 코드 (NEW)

```c
OSA_mutexLock();
if ( !fopen() ) { OSA_mutexUnlock(); return -1; }
v0 = calloc();
sprintf(); sprintf();
if ( !fopen() ) { goto LABEL_11; }
fputs();
fclose();
OSA_mutexUnlock();
return 0;
```

## 공격 시나리오

로컬 공격자가 동일 UI/RPC를 빠르게 반복 호출해 /etc/resolv.conf 류를 부분 갱신 상태로 두어 공격자 DNS 또는 빈 파일을 유도.

## 수정 설명
