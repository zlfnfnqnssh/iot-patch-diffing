# P-021

- **status**: active (v1)
- **공식**: `procfs_read` → `null_check` 부재 → `kernel_printf_deref`
  - source_detail=`/proc/mi_common/debug_func show callback seq_file`, sink_detail=`seq_printf dereferences *(const char **)(a1 + 96) without NULL check`
- **severity_hint**: medium
- **origin**: `mi_common.ko/debug_func_show`  (언어: decompiled_c)

## 요약

debug_func_show가 NEW에서 seq_file 구조체의 +96 오프셋 포인터를 seq_printf로 직접 역참조. NULL/해제 후 참조 가능성.

## 상세 설명

2.840 버전에서 debug_func_show가 debug_file_show 위임을 중단하고 직접 seq_printf를 호출한다. a1+96은 seq_file->private이거나 파일 ops data이며, proc_create_data(debug_func, ...)로 전달된 &debug_func static 변수 주소 후 +96 오프셋의 문자열 포인터를 읽는다. 초기화 경로가 모든 케이스에서 NULL 아니면 안전하나, 경합/초기화 미완 상태에서 NULL deref 가능.
