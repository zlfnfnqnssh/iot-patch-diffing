## 요약

스택 버퍼에 sprintf로 cp 명령 조립 후 system() 실행하던 경로를 snprintf + 크기 상수로 교체.

## 상세 설명

AEW 설정 저장 함수. 현재 소스 인자가 하드코딩된 경로라 즉시 악용은 불가능하지만, sprintf+system 조합 자체가 위험 시그니처. 향후 경로가 변수화되면 버퍼 오버플로 + 명령 주입이 동시 가능해지는 구조를 snprintf 크기 상한과 함께 조기 방어.

## 취약 코드 (OLD)

```c
char v1[72];
memset(v1, 0, 0x40u);
sprintf(v1, "cp %s %s", "/var/tmp/aew_para.zip", "/mnt/mtd/3A/aew_para.zip");
system(v1);
```

## 수정된 코드 (NEW)

```c
char s[80];
memset(s, 0, 0x40u);
snprintf(s, 0x3Fu, "cp %s %s", "/var/tmp/aew_para.zip", "/mnt/mtd/3A/aew_para.zip");
system(s);
```

## 공격 시나리오

현 리터럴 조합에서는 악용 불가. 그러나 향후 경로가 런타임 파생 값으로 교체되면 포맷 결과가 72바이트 버퍼를 초과하며 스택 손상 + 조립된 명령 문자열이 조작될 수 있음.

## 수정 설명

sprintf → snprintf(0x3Fu)로 크기 상한을 버퍼 크기에 고정.
