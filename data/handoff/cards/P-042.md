## 요약

zip 아카이브 디렉터리 생성 함수의 mkdir 인자 구조가 변경됨. 시그니처 축소로 모드 계산 경로가 제거.

## 상세 설명

zzip_createdir의 인자 시그니처가 (int,int,int)에서 (int,char)로 축소되고, mkdir 호출에서 a2<<27 기반 추가 플래그 계산이 제거됨. 썽크 이중 호출 j_j_*가 단일 j_*로 정규화. 디컴파일 결과만으로는 보안 영향 확정 불가하나 파일시스템 권한 모드 처리 변경 가능성 있어 사람 검토 필요.

## 취약 코드 (OLD)

```c
if ((a2 & 0x10) == 0)
  return j_j_zzip_dir_creat();
if (mkdir(a1, a2, a3, a2 << 27) == -1
    && *(_DWORD *)_errno_location() != 17)
  return 0;
return j_j_zzip_opendir();
```

## 수정된 코드 (NEW)

```c
if ((a2 & 0x10) == 0)
  return j_zzip_dir_creat();
if (mkdir() == -1
    && *(_DWORD *)_errno_location() != 17)
  return 0;
return j_zzip_opendir();
```

## 공격 시나리오

악성 zip 파일에 특수 디렉터리 엔트리를 포함시켜 구 버전에서 예상치 못한 모드 비트로 디렉터리가 생성되는 시나리오가 이론적으로 가능.

## 수정 설명

mkdir 인자 구성이 단순화되어 모드 처리가 정규화. 정확한 효과는 바이너리 수준 추가 검증 필요.
