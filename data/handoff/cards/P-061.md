## 요약

PCM FD 오픈 직후 fcntl 반환값 검증이 추가되어 실패 시 FD 누수 없이 종료.

## 상세 설명

tinyalsa 라이브러리의 PCM open 내부 래퍼. NEW에서는 open() 직후 fcntl()을 선행 호출하고 그 반환값을 검사하는 단계를 추가, 이후 기존 (a3 & 0x10) 검사와 결합하여 플래그 설정 실패를 더 세밀히 포착.

## 취약 코드 (OLD)

```c
v10 = open();
if ( v10 >= 0 )
{
  if ( (a3 & 0x10) != 0 || (fcntl(), fcntl() >= 0) )
  { /* store FD */ }
  else { close(); free(v9); return -19; }
}
```

## 수정된 코드 (NEW)

```c
v10 = open();
if ( v10 >= 0 )
{
  fcntl();
  if ( fcntl() >= 0 && ((a3 & 0x10) != 0 || (fcntl(), fcntl() >= 0)) )
  { /* store FD */ }
  else { close(); free(v9); return -19; }
}
```

## 공격 시나리오

로컬 프로세스가 ulimit 등으로 fcntl 설정을 실패시키면 OLD는 일부 경로에서 반환값 미검증으로 FD가 이상 상태로 저장될 수 있음.

## 수정 설명

fcntl >= 0 검사를 OR 체인에서 AND 선행 검사로 승격하여 양쪽 fcntl 결과를 모두 확인.
