- **status**: active (v1)
- **공식**: `config_field` → `path_normalize` 부재 → `file_open`
  - source_detail=`motorPtz device path`, sink_detail=`OSA_fileOpen on /dev path`

- **origin**: `libpdi.so/createPanTiltLocate`  (언어: decompiled_c)

## 요약

PanTilt 초기화에서 /dev/motorPtz 하드코딩 경로 strcpy 및 snprintf 대체 경로가 제거됨. access() 기반 선택 로직 삭제.

## 상세 설명

createPanTiltLocate OLD는 /dev/motorPtz 경로를 스택에 strcpy한 뒤 access 실패 시 snprintf로 대체 경로를 조립. NEW는 경로 관련 로직 전부 제거하고 PTZ flag 질의만 유지. 경로 조립 소스가 외부 입력 여부 확인 불가.

## 취약 코드 (OLD)

```c
strcpy(v9, "/dev/motorPtz");
memset(&v9[14], 0, 36);
...
if (access()) {
  snprintf();
  if (access()) goto LABEL_9;
}
```

## 수정된 코드 (NEW)

```c
j_PDI_getPeripheralPtzflag();
memset(v8, 0, 44);
if (j_PDI_chnMapGetInfoEx()) OSA_logWriteV2();
if (access()) goto LABEL_8;
```

## 공격 시나리오

대체 경로 snprintf 포맷 인자가 외부 영향 받는다면 path traversal 가능성이 있었을 수 있으나 스니펫만으로는 확정 불가.

## 수정 설명

장치 경로 의존 초기화를 PTZ flag 질의로 추상화하여 경로 조립 경로 자체 제거.
