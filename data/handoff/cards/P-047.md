# P-047

- **status**: active (v1)
- **공식**: `http_body` → `metachar_filter` 부재 → `shell_exec`
  - source_detail=`JSON method=setCloudServerType, params.server_type`, sink_detail=`system("cloudMode.sh %s") with whitelist`
- **severity_hint**: medium
- **origin**: `uhttpd/uh_slp_proto_request`  (언어: decompiled_c)

## 요약

NEW 분기에서 인증 미완 세션이 접근 가능한 setCloudServerType이 server_type을 화이트리스트 통과 후 shell 스크립트 인자로 system 호출. 화이트리스트 자체가 유일 방어선.

## 상세 설명

uhttpd가 인증 토큰(stok) 없이 들어온 '/' root 요청의 JSON body를 handle_unauthenticated_case에서 파싱할 때 setCloudServerType이라는 신규 method를 인식. 이 method는 params.server_type을 읽어 off_411270 테이블의 허용 문자열 리스트와 strcmp 순회 비교, 일치하는 테이블 리터럴(v34)만 snprintf로 쉘 커맨드에 삽입해 system() 실행. 취약 자체는 아니지만 인증 없이 system 진입점이 신규로 노출된 점을 기록해야 Hunter 단계에서 변종(예: 화이트리스트 우회/테이블 수정/TOCTOU) 추적 가능.

## 공격 시나리오

공격자가 인증 없이 POST / 바디로 {method:'setCloudServerType', params:{server_type:'<type>'}}를 전송. 화이트리스트에 존재하는 문자열만 허용되지만, 추후 항목 추가 시 메타문자가 들어있거나, 화이트리스트 조회 함수가 우회된다면 cloudMode.sh 경유 command injection 위험.

## 수정 설명

화이트리스트 배열 순회로 입력을 검증하고, strcmp 일치 리터럴만 system 인자로 사용. snprintf(v56,64,...)로 길이 경계도 설정.
