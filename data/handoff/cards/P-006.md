## 요약

52바이트 스트라이드 배열에 strlen 기반 memcpy로 문자열 기록. 대상 슬롯 최대 길이와 src strlen 비교 없음.

## 상세 설명

OLD에서는 파일을 읽어 128바이트 고정 블록에 복사하던 로직이, NEW에서는 구조체 엔트리(52바이트 stride)에 가변 길이 문자열을 누적 복사하는 형태로 바뀜. strlen 값이 52 이상이면 인접 슬롯을 침범.

## 취약 코드 (OLD)

```c
stream = fopen(a1,"r");
while (fgets(haystack,128,stream)) {
  memcpy(&v8[128*v15++], haystack, 0x80u);
}
```

## 수정된 코드 (NEW)

```c
v44 = strlen(v7);
memcpy(&a1[1][52 * v43], v7, v44);
v50 = strlen(v7);
memcpy(&a1[1][52 * v49], v7, v50);
```

## 공격 시나리오

config_tool이 처리하는 설정 엔트리 중 src 문자열이 52자 이상이면 다음 슬롯의 필드들을 덮어씀(heap/BSS 영역). 로컬 설정 파일 수정 권한이 있는 공격자가 메타데이터 변조 가능.

## 수정 설명

길이 검사를 추가하지 않고 strlen 그대로 사용 — 설계 가정이 '호출자가 미리 길이 검사' 일 수 있으나 스니펫에서 확인 불가.
