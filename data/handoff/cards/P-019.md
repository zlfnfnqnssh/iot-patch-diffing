# P-019

- **status**: active (v1)
- **공식**: `procfs_write` → `length_check` 부재 → `struct_strcpy`
  - source_detail=`dump_out path arg`, sink_detail=`strcpy into 128-byte field inside 412-byte struct element without bound`
- **severity_hint**: low

## 요약

ProcDumpOut이 /proc debugfs 경로 문자열을 stDbgFsChnInfo의 128바이트 필드에 _memzero+strcpy로 복사. 길이 검증 없음. 2.840 버전에서도 동일.

## 상세 설명

디버그 FS /proc 명령 dump_out이 채널별로 128바이트 경로 버퍼에 사용자 문자열을 strcpy. _memzero로 선제로 0으로 채운 후 bound 없이 복사, 128바이트 초과 시 인접 채널 정보 덮어쓰기.

## 취약 코드 (OLD)

```c
_memzero((char *)&stDbgFsChnInfo + 412 * v14 + 144, 128);
strcpy((char *)&stDbgFsChnInfo + 412 * v14 + 144, *(const char **)(a5 + 12));
```

## 수정된 코드 (NEW)

```c
_memzero(&dword_1A220[103 * v16 + 36], 128);
strcpy(&dword_1A220[103 * v16 + 36], *(_DWORD *)(a5 + 12));
```

## 공격 시나리오

root가 /proc 명령으로 130바이트 이상 경로 쓰기 → 인접 stDbgFsChnInfo 필드 파괴 → 후속 dump/encode 경로에서 악용.

## 수정 설명
