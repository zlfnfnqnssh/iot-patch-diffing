# P-012

- **status**: active (v1)
- **공식**: `http_header` → `bounds_check` 부재 → `stack_buffer_copy`
  - source_detail=`Authorization`, sink_detail=`strncpy(nonce, attacker_val, 0x31)`
- **severity_hint**: medium
- **origin**: `cet/http_digest_auth_check`  (언어: decompiled_c)

## 요약

HTTP Authorization 헤더 기반 Digest 검증 함수에서 nonce 복사 상한(0x31) 및 nonce count 재사용 가드(playback attack) 추가.

## 상세 설명

HTTP 인증 처리기에서 Authorization 헤더의 nonce를 48바이트 버퍼에 복사할 때 크기 0x31(49) 상한을 명시하고 null termination으로 overflow를 차단. 추가로 nonce count(nc) 역행 시 이전 접수값보다 작거나 같으면 재사용 공격으로 판정하여 거부.

## 공격 시나리오

공격자가 nonce 헤더를 49바이트 이상 삽입하거나 이미 사용한 nc 값을 재제출하여 세션 재현 공격 시도.
