# P-023

- **status**: active (v1)
- **공식**: `rpc_arg` → `length_bound` 부재 → `heap_buffer_copy`
  - source_detail=`RX8010_write n 인자 (내부 호출)`, sink_detail=`memcpy(dword_2FE10 + 1, src, n)`
- **severity_hint**: low
- **origin**: `pdc.ko/RX8010_readRtc`  (언어: decompiled_c)

## 요약

I2C write 내부 루틴의 길이 n이 고정 힙 버퍼(144B) 범위를 넘을 때 memcpy 전에 `n > 0x8F` 검사 추가. OLD는 검사 없이 바로 memcpy.

## 상세 설명

RX8010 I2C RTC 드라이버에서 write helper가 내부 정적 힙 버퍼(dword_2FE10, 144B)에 사용자 지정 길이 n만큼 memcpy. OLD에는 n 상한 검사가 없어 호출자가 큰 n을 넘기면 힙 오버플로우 가능. NEW는 `n > 0x8F`(143) 상한 검사로 차단.

## 취약 코드 (OLD)

```c
v19 = 0;
if ( RX8010_write(dword_27F9C, (int)v15) )
{
    OSA_logWrite(0, 1, "[pdc] ERROR  ... Fail to read RTC register\n", "RX8010_readRtc", 300);
    return -1;
}
```

## 수정된 코드 (NEW)

```c
if ( n > 0x8F )
{
    OSA_logWrite(0, 1, "[pdc] ERROR  ... The write data is too long\n", "RX8010_write", 238);
}
else
{

    *(_BYTE *)dword_2FE10 = v21;
    memcpy((void *)(v15 + 1), src, n);
}
```

## 공격 시나리오

로컬 루트가 RTC proc 인터페이스나 ioctl로 비정상적으로 긴 I2C 명령 페이로드를 내부 경로로 전달할 때 OLD 경로는 힙 오버플로우. 공격 표면은 커널 내부라 원격성 낮음.

## 수정 설명

memcpy 전에 상한 검사 추가. 경계값 0x8F는 버퍼 시작 바이트(v21)를 제외한 143바이트에 대응.
