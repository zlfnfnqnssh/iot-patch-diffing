# P-003

- **status**: active (v1)
- **공식**: `net_packet` → `length_bound` 부재 → `heap_buffer_copy`
  - source_detail=`asn.1 encoded length field`, sink_detail=`memcpy with length from ASN.1 LV field`

## 요약

ASN.1 인코더가 오브젝트 길이를 dereference해 memcpy. 상한 비교는 있으나 length 필드 신뢰 전제. 리팩토링 상 간접접근 경로 변경.

## 상세 설명

ASN.1 인코더에서 TLV 길이 필드를 memcpy 크기로 사용. 상한 a4와 비교하므로 명시적 OOB는 없으나, 네트워크 경계에서 유입된 ASN.1 구조체를 파싱할 경우 길이 악용 가능성 남음.

## 취약 코드 (OLD)

```c
sub_178DC(v12, 115, 0);
v13 = *(_DWORD *)a2;
if ( *(_DWORD *)a2 <= a4 )
  memcpy(a3, *(const void **)(a2+4), *(_DWORD *)a2);
else
  puts("encode_len too small");
```

```c
v15 = a2;
sub_1549C(v12, 115, 0, &v15);

```

## 공격 시나리오

공격자가 조작된 ASN.1 객체의 length 필드를 0에 가깝게 혹은 음수로 설정해 memcpy 동작 왜곡 시도. 현재 코드는 unsigned 비교라 저지되지만 서명 함수 연쇄에 따라 영향 가능.
