## 요약

i2cOpen의 고정 크기 스택 버퍼에 `/dev/i2c-%s` 포맷 sprintf. 버퍼 크기 64→68로 소폭 확장됐으나 snprintf 치환은 없음.

## 상세 설명

/dev/i2c-<bus> 경로 조합에 sprintf 사용. i2cBus가 호출측 argv/환경에서 들어올 수 있어 긴 문자열이면 스택 오버플로우. 버퍼 4바이트 확장은 근본 수정이 아니며 snprintf 도입이 필요.

## 취약 코드 (OLD)

```c
char v1[64];
sprintf(v1, "/dev/i2c-%s", (const char *)i2cBus);
result = open(v1, 2);
```

## 수정된 코드 (NEW)

```c
char s[68];
sprintf(s, "/dev/i2c-%s", (const char *)i2cBus);
result = open(s, 2);
```

## 공격 시나리오

로컬에서 i2c_read_write 실행 시 bus 인자를 60바이트 이상 문자열로 넘기면 /dev/i2c-... 포맷 결과가 버퍼를 초과해 스택 오버플로우.

## 수정 설명

버퍼 크기만 증가. sprintf → snprintf 치환이 필요했으나 하지 않음.
