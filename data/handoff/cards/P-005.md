  - source_detail=`log level color idx`, sink_detail=`color table index with +10 offset`

## 요약

로그 컬러 테이블 인덱싱이 `s_levelColor[color]`에서 `off_28EF0[a2+10]`로 변경. 경계 검사 없는 전역 포인터 배열 색인 유지.

## 상세 설명

vsnprintf는 크기 0x1FF로 제한되어 로컬 버퍼 오버플로우 방어 유지. 다만 컬러 선택 배열 `off_28EF0`을 `a2+10` 인덱스로 접근하므로 a2가 큰 값이면 전역 OOB 읽기 가능. a2는 내부 로그 레벨 파라미터라 실제 공격 경로는 제한적.

## 취약 코드 (OLD)

```c
char buf[528];
memset(buf,0,0x200u);
vsnprintf(buf,0x1FFu,pFmta,varg_r3);
printf("%s%s%s", s_levelColor[color], buf, "\x1B[0m");
```

## 수정된 코드 (NEW)

```c
char s[536];

vsnprintf(s,0x1FFu,a3,varg_r3);

```

## 공격 시나리오

OSA_logWrite 호출자가 level 파라미터에 범위 밖 값을 넘기면 전역 포인터 배열 OOB 읽기로 크래시/정보 유출 가능.

## 수정 설명

컬러 테이블 기준점을 +10 오프셋으로 이동. 경계 검사는 추가되지 않음.
