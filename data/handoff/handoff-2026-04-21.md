## 6. Agent 프롬프트 템플릿 (Zero-Day 블라인드, v3)

```
You are a Zero-Day Hunter Agent (A{i}).

Working directory: D:\Task\4\project

## Read FIRST
1. .claude/skills/stage2/prompts/zero_day_hunter.md
2. .claude/skills/stage2/rules/hard-rules.md
3. docs/pattern-card-spec.md

## STRICT CONSTRAINTS
- ZERO external knowledge. No CVE lookups.
- MUST NOT read cve-*.md, kve*.md, advisory*, changelog*.
- No WebSearch / WebFetch.
- Output raw_reasoning MUST NOT contain CVE-\d{4}-\d+ or KVE-\d{4}-\d+.

## EXTRA REQUIREMENT — cite card similarity + trace data flow

For every verdict your `root_cause` MUST include:
1. Name at least one card_id (e.g., "P-044") and which axis matches vs differs.
2. Cite concrete pseudocode (specific line or token).

## CRITICAL: size-literal anchoring 금지

A size/length argument being present or a const literal does NOT prove safety.
Trace size origin:
- 0x7FF, 256, sizeof(buf) → literal, bounded
- strlen(user_input) → attacker-controlled, unbounded
- (p2 - p1) where p1/p2 from strchr/strstr on attacker input → HIGH RISK
- (end - start) where start = getHeader(...) + end = strchr(start, ...) → HIGH RISK

When size provenance unclear → conf 0.5-0.65 + needs_human_review=true.

## Task

Input: tmp/zd/in_r1_b{N}_a{i}.json (functions + pattern_cards context, P-106 hidden).
Output: JSON ARRAY → tmp/zd/in_r1_b{N}_a{i}_out.json. agent_id="A{i}".

## Report

Under 250 words: processed/vuln/benign, confidence distribution, top 3 with card citation, forbidden-file temptations resisted.
```

**Agent 호출 시 주의**:
- `subagent_type=general-purpose`, `model=opus`
- 4개 다 같은 메시지에서 동시 spawn (병렬 보장)

## 8. 알려진 함정 / 주의사항

1. **`card_id` vs `pk` 혼동**: `card_id`는 텍스트 라벨 ("P-106"), `pk`는 INT primary key (74). `--exclude-card-pk` 같은 CLI 옵션은 pk를 받음. DB FK도 전부 pk 기반.

2. **웹 서버 포트 8787 중복**: 이미 떠있을 때 `python app.py` 치면 `WinError 10048`. `netstat -ano | grep :8787` + `taskkill` 로 kill 후 재시작.

3. **FastAPI TemplateResponse 서명**: 최신 버전은 `templates.TemplateResponse(request, "file.html", {...})` — 기존 dict 통째로 넘기는 방식 X.

4. **IDA crash 복구**: `extract_all_funcs.py` 는 `IDA_RESUME=1` (기본) + `.partial` 체크포인트로 자동 이어받음. 중간에 죽어도 다시 실행하면 됨.

5. **Stripped binary prefilter**: sonia처럼 심볼 없는 C++ 바이너리는 `strncpy(` 같은 keyword 매치 안 됨. `prefilter.py` 의 DANGEROUS_KEYWORDS에 HTTP 헤더 literal (`"Host"`, `"Cseq"`) + 셸 명령 literal 포함시켜 해결.

6. **pseudocode 캐싱 이슈 없음**: DB 저장된 pseudocode + strings는 불변. 단 IDA 재분석 시 주소 변화할 수 있으므로 `zero_day_functions` 는 run별로 독립.

7. **size-literal anchoring (v3 프롬프트의 핵심 교훈)**: Agent가 pseudocode에서 literal size 인자를 보면 bounded 로 착각하기 쉬움. 실제 CVE는 size 자체가 attacker-derived 계산 결과인 경우가 많음. 프롬프트에서 size origin 추적 강제 필요.

8. **블라인드 제약 검증**: 매 batch 후 `raw_reasoning`에 `CVE-\d{4}-\d+` 정규식 매치 0건 확인. 1건이라도 누출되면 해당 verdict 무효 + Agent 프롬프트 재검토.
